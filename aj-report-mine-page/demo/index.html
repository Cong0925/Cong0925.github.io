🗄️ 步骤一：创建数据库连接

使用 Navicat 16 创建一个名为 localhost 的本地连接

连接配置信息

主机 localhost
端口 3306
用户 root
密码 root

📝 步骤二：修改启动脚本

通过代码编辑软件打开拉取的项目根目录，修改以下文件

📄 根目录 → report-core → src → main → assembly → bin → start.bat

将 JAVA_HOME 改为自己 Java 的环境变量地址

start.bat 
# 修改前
set JAVA_HOME=C:\Program Files\Java\jdk1.8.0_xxx

# 修改后（改成你自己的 JDK 安装路径）
set JAVA_HOME=C:\Program Files\Java\jdk1.8.0_301

⚙️ 步骤三：修改数据库配置

📄 根目录 → report-core → src → main → resources → bootstrap.yml

将配置改为自己的 MySQL 设置

bootstrap.yml 
spring:
  datasource:
    # 数据库连接地址
    url: jdbc:mysql://localhost:3306/report?useUnicode=true&characterEncoding=utf-8&useSSL=false&serverTimezone=Asia/Shanghai
    # 数据库用户名
    username: root
    # 数据库密码
    password: root
    # 驱动类名
    driver-class-name: com.mysql.cj.jdbc.Driver
⚠️

请确保 MySQL 8.0 已安装并启动，并且 root 用户密码为 root（或修改为你自己的密码）

06

常见问题

❓ 环境变量如何配置？ +

右键"我的电脑" → 属性 → 高级系统设置 → 环境变量

在"系统变量"中添加：

  • JAVA_HOME: JDK 安装路径，如 C:\Program Files\Java\jdk1.8.0_301
  • MAVEN_HOME: Maven 安装路径，如 C:\apache-maven-3.6.3

然后在 Path 变量中添加 %JAVA_HOME%\bin 和 %MAVEN_HOME%\bin

❓ MySQL 连接失败怎么办？ +

请检查以下几点：

  • MySQL 服务是否已启动（在服务管理器中查看）
  • 用户名和密码是否正确（默认都是 root）
  • 端口 3306 是否被占用
  • 是否使用 MySQL 8.0 版本
  • 数据库是否已创建（数据库名应为 report）
❓ Node.js 版本不兼容？ +

使用 NVM 安装指定版本：

CMD 
# 安装 Node.js 14.16.0
nvm install 14.16.0

# 切换到该版本
nvm use 14.16.0

# 验证版本
node -v
❓ Maven 打包失败？ +

可能的原因和解决方案：

  • 网络问题：配置 Maven 阿里云镜像加速
  • JDK 版本：确保使用 JDK 1.8
  • 内存不足：增加 Maven 内存配置
  • 清理重试：执行 mvn clean install -DskipTests
❓ 前端启动后页面空白？ +

请检查：

  • 后端服务是否已启动
  • 浏览器控制台是否有报错信息
  • 代理配置是否正确
  • 尝试清除浏览器缓存后刷新
