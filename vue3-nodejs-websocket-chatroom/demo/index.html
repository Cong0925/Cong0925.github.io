验证安装

打开终端或命令提示符，运行以下命令验证环境：

终端 
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version

# 检查 Git 版本
git --version

第一步：克隆项目

首先，使用 Git 克隆项目到本地：

终端 
# 克隆项目
git clone https://github.com/Cong0925/vue3_nodejs_websocket_chatRoom.git

# 进入项目目录
cd vue3_nodejs_websocket_chatRoom

如果网速较慢，可以使用 Gitee 镜像或下载 ZIP 包。

第二步：安装依赖

安装前端项目依赖：

终端（项目根目录） 
# 安装前端依赖
npm install

加速安装（可选）

如果安装速度较慢，可以使用淘宝镜像：

终端 
# 设置淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install

第三步：启动WebSocket服务器

打开新的终端窗口，启动WebSocket服务器：

终端（新窗口） 
# 进入服务器目录
cd server

# 启动WebSocket服务器
node index.js

启动成功后，终端会显示：websocket open

WebSocket服务器默认监听 8000 端口。

第四步：启动前端

打开另一个终端窗口，启动前端开发服务器：

终端（新窗口） 
# 启动前端开发服务器
npm run serve

启动成功后，终端会显示访问地址，通常是 http://localhost:8080

第五步：开始聊天

现在可以开始使用聊天室了！

1

打开浏览器

访问 http://localhost:8080

2

登录

输入用户名（至少6位）和密码（密码与用户名相同）

3

多开窗口测试

打开多个浏览器窗口或标签页，使用不同用户名登录

4

发送消息

在任一窗口输入消息，其他窗口会实时收到

提示：登录后，用户信息会保存在本地存储中，下次访问会自动登录。

常见问题

端口被占用怎么办？

如果8000端口被占用，可以修改 server/index.js 中的端口号：

// 修改端口号
const server = new Ws.Server({port: 8001})  // 改为其他端口

同时需要修改前端配置文件 src/config/index.ts 中的API地址。

前端无法连接WebSocket服务器？

请检查以下几点：

  • WebSocket服务器是否已启动（终端显示 websocket open）
  • 服务器端口是否正确（默认8000）
  • 是否有防火墙阻止了连接
  • 浏览器控制台是否有错误信息
如何修改登录验证逻辑？

登录验证逻辑在 src/views/login/login.vue 中，当前逻辑是：

  • 用户名长度不小于6位
  • 密码必须与用户名相同

你可以根据需要修改 goPage() 函数中的验证逻辑。

如何添加更多功能？

你可以参考以下方向扩展功能：

  • 添加用户列表显示
  • 实现私聊功能
  • 添加消息类型（图片、表情等）
  • 实现消息持久化存储
  • 添加用户头像和昵称
