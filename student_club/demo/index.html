管理后台

admin.student_club

端口：:2807

  • 活动创建与审核

  • 内容发布管理
  • 数据统计分析

www.student_club

  • 活动信息浏览

🚀 快速开始

按照以下步骤快速启动项目

1

克隆项目

从 GitHub 仓库克隆项目代码到本地：

git clone https://github.com/Cong0925/student_club.git
cd student_club
2

安装管理后台依赖

进入管理后台目录并安装依赖：

cd admin.student_club
npm install
3

安装前台网站依赖

进入前台网站目录并安装依赖：

cd ../www.student_club
npm install
4

启动管理后台

在管理后台目录下启动开发服务器：

cd admin.student_club
npm run dev

启动后访问：http://localhost:2807

5

启动前台网站

在前台网站目录下启动开发服务器：

cd www.student_club
npm run dev

启动后访问：http://localhost:5173

6

快速启动脚本（可选）

如果项目根目录提供了快速启动脚本，可以一键安装所有依赖：

# 在项目根目录执行
npm install

# 分别启动管理后台和前台网站（可使用不同的终端窗口）
npm run dev:admin    # 启动管理后台
npm run dev:www      # 启动前台网站

提示：如果根目录没有 package.json，请按步骤 2-5 分别进入子目录操作。

🗃️ Mock 数据配置

在后端服务未就绪时，使用 Mock 数据进行前端开发调试

📝 Mock 数据说明

本项目支持通过 Mock 数据进行前端独立开发。Mock 数据通常配置在以下位置：

  • src/mock/ - Mock 数据目录
  • src/api/mock.js - Mock 接口拦截配置

🔌 启用 Mock 数据

// 在 main.js 或专门的 mock 配置文件中
// 开发环境启用 Mock
if (import.meta.env.DEV) {
    // 注册 Mock 服务
    import('./mock/index.js');
}

// 或者通过环境变量控制
// .env.development
VITE_USE_MOCK=true

用户注册与登录

功能说明

支持用户名/密码注册、邮箱验证、密码找回、自动登录等功能。提供管理员和普通用户两种登录入口，不同角色登录后展示不同的管理界面。

操作步骤

  1. 访问前台网站首页，点击右上角「注册」按钮
  2. 填写用户名、邮箱、密码等信息，完成注册
  3. 使用注册的账号密码登录系统
  4. 管理员通过后台登录入口（:2807）进入管理界面
📸

登录界面截图占位符

代码示例

// 登录请求示例
const login = async (credentials) => {
    const response = await api.post('/auth/login', {
        username: credentials.username,
        password: credentials.password
    });
    // 存储 token
    localStorage.setItem('token', response.data.token);
    return response.data;
};

社团创建与管理

功能说明

支持社团创建、编辑、删除、分类管理。社团信息包括名称、简介、分类标签、封面图片、成员上限等。支持社团状态控制（正常/暂停/解散）和申请审核流程。

操作步骤

  1. 管理员在后台进入「社团管理」模块
  2. 点击「创建社团」按钮，填写社团基本信息
  3. 设置社团分类标签和封面图片
  4. 提交创建申请，等待审核通过
  5. 社团创建成功后，可在前台网站展示
📸

社团管理界面截图占位符

代码示例

// 创建社团
const createClub = async (clubData) => {
    const response = await api.post('/clubs', {
        name: clubData.name,
        description: clubData.description,
        category: clubData.category,
        coverImage: clubData.coverImage,
        memberLimit: clubData.memberLimit
    });
    return response.data;
};

活动发布与报名

功能说明

支持活动创建、编辑、发布、报名管理、签到记录、活动回顾。活动状态流转包括：草稿 -> 报名中 -> 进行中 -> 已结束。支持线上线下活动，可设置报名人数上限。

操作步骤

  1. 社团管理员在后台进入「活动管理」模块
  2. 点击「创建活动」，填写活动名称、时间、地点等信息
  3. 设置报名截止时间和人数上限
  4. 发布活动后，前台网站展示活动信息
  5. 用户可在前台网站点击「报名参加」
  6. 活动开始后，管理员可进行签到管理
📸

活动管理界面截图占位符

代码示例

// 活动报名
const joinActivity = async (activityId) => {
    const response = await api.post(`/activities/${activityId}/join`);
    if (response.data.success) {
        // 报名成功，更新 UI
        showToast('报名成功！请准时参加活动');
    }

成员管理

支持成员信息管理（姓名、学号、联系方式）、角色权限分配（管理员/社长/普通成员）、入团申请与审核、成员数据统计与导出。

操作步骤

  1. 用户在前台网站申请加入社团
  2. 社团管理员在后台审核入团申请
  3. 审核通过后，成员信息自动录入系统
  4. 管理员可设置成员角色和权限
  5. 支持导出成员信息为 Excel 文件
📸

代码示例

// 获取社团成员列表
const getMembers = async (clubId, page = 1) => {
    const response = await api.get(`/clubs/${clubId}/members`, {
        params: { page, pageSize: 20 }

通知公告

功能说明

支持通知公告的创建、编辑、发布、置顶、删除。公告类型包括系统公告、社团公告、活动通知等。支持富文本编辑，可插入图片和链接。

操作步骤

  1. 管理员在后台进入「内容管理」->「通知公告」模块
  2. 点击「发布公告」，填写标题和内容
  3. 选择公告类型和发布范围
  4. 支持设置置顶和有效期
  5. 发布公告后，前台网站展示通知信息
📸

通知公告管理界面截图占位符

代码示例

// 发布公告
const publishNotice = async (noticeData) => {
    const response = await api.post('/notices', {
        title: noticeData.title,
        content: noticeData.content, // 富文本 HTML
        type: noticeData.type,       // system | club | activity

数据统计

功能说明

提供社团数据可视化、活动参与率统计、成员增长趋势图、数据导出为 Excel 等功能。帮助管理者快速了解社团运营状况，做出数据驱动的决策。

操作步骤

  1. 管理员在后台进入「数据统计」模块
  2. 查看社团数据总览（成员数、活动数、文章数等）
  3. 查看活动参与率统计图表
  4. 查看成员增长趋势折线图
  5. 点击「导出数据」下载 Excel 报表
📸

数据统计界面截图占位符

代码示例

// 获取统计数据
const getStatistics = async (clubId) => {
    const response = await api.get(`/clubs/${clubId}/statistics`);
    return {
        memberCount: response.data.memberCount,
        activityCount: response.data.activityCount,
        articleCount: response.data.articleCount,
        memberGrowth: response.data.memberGrowthTrend, // 近30天
        activityParticipation: response.data.participationRate
    };
};

// 导出 Excel
const exportData = async (clubId, type) => {
    const response = await api.get(`/clubs/${clubId}/export`, {
        params: { type }, // members | activities
        responseType: 'blob'
    });
    // 触发下载
    const url = window.URL.createObjectURL(response.data);
    const link = document.createElement('a');
    link.href = url;
    link.download = `${type}_export.xlsx`;
    link.click();
};

❓ 常见问题 FAQ

遇到问题时请先查阅以下常见解答

npm install 安装依赖失败怎么办？
  1. 检查网络连接，尝试使用淘宝镜像：npm config set registry https://registry.npmmirror.com
  2. 删除 node_modules 和 package-lock.json 后重新安装
  3. 检查 Node.js 版本是否满足要求（>= 14.0.0）
  4. 尝试使用 npm install --legacy-peer-deps
启动开发服务器后页面空白？
  1. 检查浏览器控制台是否有报错信息
  2. 确认 .env 文件配置正确
  3. 确认后端 API 服务已启动或 Mock 数据已启用
  4. 尝试清除浏览器缓存后刷新页面
端口被占用（2807 或 5173）？
  1. Windows: netstat -ano | findstr :2807 查找占用进程，然后 taskkill /PID <进程号> /F
  2. Mac/Linux: lsof -i :2807 查找，kill -9 <进程号>
  3. 或者修改 vite.config.js 中的 server.port 配置使用其他端口
如何切换 Mock 数据和真实 API？
  1. 在 .env.development 中设置 VITE_USE_MOCK=true 启用 Mock
  2. 设置 VITE_USE_MOCK=false 并配置正确的 VITE_API_BASE_URL 使用真实 API
  3. 修改配置后需要重启开发服务器
Element Plus 样式不生效？
  1. 确认已正确安装 Element Plus：npm install element-plus
  2. 在 main.js 中正确引入并注册：app.use(ElementPlus)
  3. 检查是否使用了按需引入方式，确认 unplugin-vue-components 配置正确
  4. 尝试完整引入方式排查问题
如何参与项目贡献？
  1. Fork 项目仓库到你的 GitHub 账号
  2. 创建功能分支：git checkout -b feature/your-feature
  3. 完成开发后提交代码并发起 Pull Request
  4. 遵循项目的代码规范和提交规范
