01

安装 Hugo

1.1 选择安装方式

必须安装 Hugo Extended 版本，普通版本不支持 SASS 编译。根据你的操作系统选择对应的安装方式：

方法一：Chocolatey（推荐） 
choco install hugo-extended
方法二：Scoop 
scoop install hugo-extended
方法三：手动下载 
1. 访问 https://github.com/gohugoio/hugo/releases
2. 下载 hugo_extended_*_windows-amd64.zip
3. 解压文件，将 hugo.exe 复制到系统 PATH 目录
提示：推荐使用 Chocolatey 或 Scoop 包管理器， 可以自动处理依赖和版本更新。
Homebrew 
# 安装 Hugo
brew install hugo

# 如需安装最新版本
brew upgrade hugo
提示：macOS 下 Homebrew 默认安装 Hugo Extended 版本， 无需额外指定。
Snap（推荐） 
sudo snap install hugo
APT（Ubuntu/Debian） 
sudo apt install hugo
从源码编译 
# 需要先安装 Go 语言环境
go install -tags extended github.com/gohugoio/hugo@latest

1.2 验证安装

安装完成后，打开终端验证 Hugo 是否安装成功：

终端命令 
hugo version
预期输出：应该看到类似 hugo v0.128.0+extended 的输出， 注意版本号中必须包含 "extended" 字样。

终端执行 hugo version 后的输出截图

02

创建新站点

2.1 使用 Hugo 创建站点

使用 Hugo 命令行工具创建一个新的站点：

终端命令 
# 创建新站点
hugo new site my-blog

# 进入站点目录
cd my-blog

执行完毕后，Hugo 会自动生成以下目录结构：

目录结构 
my-blog/
├── archetypes/      # 文章模板
├── assets/          # 需要处理的资源文件
├── content/         # 内容目录
├── data/            # 数据文件
├── i18n/            # 国际化文件
├── layouts/         # 页面模板
├── static/          # 静态资源
├── themes/          # 主题目录
└── hugo.toml        # 站点配置文件

2.2 克隆本项目（推荐）

如果想直接使用本项目的配置和主题，可以克隆仓库：

终端命令 
# 克隆项目
git clone https://github.com/Cong0925/hugoBasicExample.git

# 进入项目目录
cd hugoBasicExample
重要：本项目已包含完整的主题和配置， 克隆后即可直接使用，无需额外安装主题。
03

创建文章

3.1 使用命令创建文章

Hugo 提供了便捷的命令来创建新文章：

终端命令 
# 创建新文章
hugo new content content/post/my-first-post.md

# 创建关于页面
hugo new content content/about.md

3.2 文章 Front Matter 格式

每篇文章的顶部都需要 Front Matter 元数据，支持 TOML、YAML 和 JSON 格式：

content/post/my-first-post.md 
---
title: "我的第一篇博客"
date: 2024-01-15
draft: false
tags: ["博客", "Hugo"]
categories: ["技术"]
series: ["Hugo 入门"]
description: "这是我的第一篇 Hugo 博客文章"
---

# 文章标题

这里是文章内容，支持 **Markdown** 语法。

## 二级标题

- 列表项 1
- 列表项 2
- 列表项 3

```go
// 代码块示例
fmt.Println("Hello, Hugo!")
```
提示：将 draft 设为 false 才会在网站上显示。 默认创建的文章为草稿状态（draft: true）。

3.3 文章管理建议

  • 文件命名使用 kebab-case 格式，如 my-first-post.md
  • 每篇文章必须包含 title 和 date 字段
  • 使用 tags 和 categories 组织文章
  • 文章内容使用标准 Markdown 语法
  • 图片建议放在 static/images/ 目录下
04

主题配置

4.1 主要配置文件说明

Hugo 采用模块化配置，将不同功能的配置拆分到独立文件中：

文件路径 说明 修改频率
config/_default/config.toml 主配置文件（站点标题、语言等） 较少修改
config/_default/params.toml 网站参数（描述、Logo、功能开关） 经常修改
config/_default/menus/menu.zh.toml 中文菜单配置 按需修改
config/_default/menus/menu.en.toml 英文菜单配置 按需修改

4.2 修改网站信息

编辑 config/_default/params.toml 文件来自定义网站信息：

config/_default/params.toml 
# 修改网站描述
introDescription = "我的个人技术博客"

# 修改 Logo（放在 static 目录下）
logo = "/images/logo.png"

# 启用搜索功能
enableSearch = true

# 深色模式设置
# false: 跟随系统  |  true: 强制浅色
enforceLightMode = false

# 文章每页显示数量
paginate = 10

4.3 修改导航菜单

编辑 config/_default/menus/menu.zh.toml 来自定义导航菜单：

config/_default/menus/menu.zh.toml 
[[main]]
  name = "主页"
  url = ""
  weight = -110

[[main]]
  name = "分类"
  url = "categories/"
  weight = -108

[[main]]
  name = "关于"
  url = "about/"
  weight = -107

菜单配置说明

weight 值越小，菜单项越靠前显示。

url 为空时表示站点根目录。

4.4 安装和切换主题

如果需要使用其他主题，可以通过以下方式安装：

终端命令 
# 方式一：Git 子模块（推荐）
git submodule add https://github.com/author/theme-name.git themes/theme-name

# 方式二：Hugo Modules（需要 Go 环境）
hugo mod get github.com/author/theme-name

安装后在 config/_default/config.toml 中启用：

config/_default/config.toml 
# 设置使用的主题
theme = "theme-name"
05

本地预览

5.1 启动开发服务器

Hugo 内置了高性能的开发服务器，支持实时预览和热重载：

终端命令 
# 启动开发服务器（默认端口 1313）
hugo server

# 启动并包含草稿文章
hugo server -D

# 指定端口启动
hugo server --port 8080

# 清除缓存并启动
hugo server --gc --ignoreCache

Hugo server 启动成功后的终端输出截图

5.2 访问网站

服务器启动后，打开浏览器访问以下地址：

访问地址

本地预览： http://localhost:1313/

网络访问： http://0.0.0.0:1313/ （局域网内其他设备可访问）

成功标志：看到博客首页即表示启动成功。 修改文件后页面会自动刷新。

5.3 常用开发命令

命令 说明
hugo server 启动开发服务器
hugo server -D 包含草稿文章一起预览
Ctrl + C 停止服务器
hugo 构建静态文件到 public/ 目录
hugo --minify 构建并压缩静态文件（生产环境）
hugo new content post/xxx.md 创建新文章
06

部署到 GitHub Pages

6.1 创建 GitHub 仓库

在 GitHub 上创建一个新的仓库，用于存放 Hugo 生成的静态文件：

  • 仓库名称格式：用户名.github.io（个人主页）或任意名称
  • 选择 Public 可见性
  • 不要初始化 README 文件

6.2 构建静态文件

在项目根目录下执行构建命令：

终端命令 
# 构建静态文件（输出到 public/ 目录）
hugo

# 构建生产版本（压缩 CSS/JS）
hugo --minify

# 查看构建结果
ls -la public/
提示：确保 .gitignore 文件中包含 public/ 目录， 避免将构建产物提交到源码仓库。

6.3 使用 GitHub Actions 自动部署

推荐使用 GitHub Actions 实现自动化部署，在项目根目录创建以下工作流文件：

.github/workflows/hugo.yml 
name: Deploy Hugo to GitHub Pages

on:
  push:
    branches:
      - main

permissions:
  contents: write

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: true

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./public

6.4 手动部署（使用 git push）

如果不使用 GitHub Actions，也可以手动部署：

终端命令 
# 构建静态文件
hugo --minify

# 进入构建目录
cd public

# 初始化 Git 仓库
git init
git add .
git commit -m "Initial deploy"

# 推送到 GitHub Pages 仓库
git remote add origin https://github.com/用户名/用户名.github.io.git
git push -u origin main
注意：手动部署需要每次修改后重新执行构建和推送。 建议使用 GitHub Actions 实现自动化。

部署完成后，访问以下地址验证：

访问地址

项目页面： https://用户名.github.io/仓库名/
